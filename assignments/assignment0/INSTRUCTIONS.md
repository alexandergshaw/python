# Assignment 0: Setup, Git, and Vercel (GUI-Only)

Welcome! In this assignment you will set up your course project using only visual GitHub tools.

## Rules for this class
- Only edit `student_code.py`.
- Use the **Codespaces editor**, **Source Control panel**, **Testing panel**, and the **GitHub website**.
- Do **not** use terminal commands.

## Goal
Change the starter variable in `student_code.py`, run tests from the Testing panel, commit/push from Source Control, and merge a pull request.

## Step-by-step workflow
1. Open the repository on GitHub and click **Fork**.
2. In your fork, click **Deploy** (Vercel button) and complete deployment.
3. Back on GitHub, click **Code → Codespaces → Create codespace on main**.
4. In Codespaces, open the **Source Control** panel and create a new branch named `assignment0-setup`.
5. Open `assignments/assignment0/student_code.py`.
6. Change `student_name` from `"Your Name"` to your name.
7. Open the **Testing** panel (beaker icon) and run `test_assignment.py`.
8. Confirm tests are green.
9. Go to **Source Control**, stage the changed file, write a commit message, and click **Commit**.
10. Click **Sync Changes** in Source Control to push.
11. On the GitHub website, open the branch comparison banner and click **Compare & pull request**.
12. Create the PR and click **Merge pull request**.

## Worked examples (use different data)
- Example A: `student_name = "Maya"`, with values `[12, 34, 8]` and labels `["Red", "Blue", "Green"]`
- Example B: `student_name = "Noah"`, with values `[5, 9, 7, 4]` and labels `["Mon", "Tue", "Wed", "Thu"]`

## What to submit
- A merged pull request showing your `student_code.py` change.
